### 创建三个线程，分别在打印自己实际休眠时间后休眠5秒钟，然后再次重复上述过程，直到被用户终止时为止。

本次作业难点所在。关于休眠五秒，和打印自己的实际休眠时间。休眠五秒还算简单，在测量自身休眠时间方面，使用获取时间戳的方式进行解决。即线程挂起后记录当前时间戳，线程开始运行时记录当前时间戳并求差值打印出来。

领悟：
------
### 最基本的线程机制
Thread和Runnable目前看起来还差不多，日后深入学习时会改进此项认知。<br>Thread类型线程需继承Thread类，命名线程名。其run，start，wait等方法可以重写。<br>Runnable类型需要实现其接口

### 关于线程的生命周期
新建状态->执行start（）方法->就绪状态->执行run（）方法->运行状态（系统调度，获取cpu等资源）->run（）方法结束->死亡状态<br>其中在运行状态时，可以使用sleep（睡眠）、suspend（挂起）等方法将其挂起，进入阻塞状态，让出cpu等资源<br>可以使用stop（）或destroy（）方法强行将进程终止

### 关于java时间戳
Java中，获取时间戳的指令为：
```
System.currentTimeMillis()
```
需要注意的是，java时间戳是long类型，而且是精确到毫秒级的。想要变成秒得经过变换:
```
((float)(System.currentTimeMillis())/1000)
```
毕竟是要求变成秒，而对于这种程序，线程之间调度导致的误差应该不会超过1秒，所以还是得先把时间戳变成float型，再除1000。
